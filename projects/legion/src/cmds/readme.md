# `legion bootstrap`

实现 `seed -> v1 -> v2` 的诚实自举编译链，支持 `CLR` 与 `WASM` 目标。

## 工作流程

1. 使用 seed（已有的可运行 `legion`）编译编译器源码，得到 v1。
2. 运行 v1 验证其可执行。
3. 使用 v1 编译同一份编译器源码，得到 v2。
4. 比对 v1 与 v2 产物字节一致性，验证确定性。

## 诚实自举护栏

- v1 必须实际读取源文件内容，输出依赖源文件。
- 禁止预编译模板、自复制、外部编译器代理、差分测试伪装。
- 禁止使用固定输入值代替真实源文件路径。

## `WASM` 目标补充

- v1 产物由 `.wasm` 和 `.mjs` 启动壳组成，通过 `node` 运行。
- 微编译器通过 `wasm_import` 从 `Node` 启动壳获取 I/O 能力。
- v1 运行时通过 `read_source_byte` 导入读取源文件，统计字节数并生成返回该计数的 `WASM` 模块。
- 自举验证要求两次读取同一源文件得到完全一致的输出。
