# 类型系统 (Type System)

Valkyrie 拥有一个高度表达且严谨的类型系统，旨在平衡开发效率、运行时性能与编译时安全性。

本章将带你由浅入深地探索 Valkyrie 类型系统的各个维度。

## 学习路线图

按照以下顺序阅读，可以获得最佳的学习体验：

### 1. 基础建模
- **[代数数据类型 (ADT)](./algebraic-data-types.md)**: 理解 `unite` 与积类型，以及 sum / product 的建模方式。
- **[联合类型 (Unite Types)](./union.md)**：深入了解 `unite`、可选 `tag` 与模式匹配的关系。
- **[指针与引用](./pointer-type.md)**: 掌握原始指针、引用修饰符以及它们与内存布局的关系。

### 2. 抽象与多态
- **[泛型编程](./generics.md)**: 学习如何编写参数化代码以及如何通过特征（Trait）添加约束。
- **[关联类型](./associated-types.md)**: 探索具名 Trait 中的类型占位符，理解它为何不属于匿名 row。
- **[行类型与多态](./row-types.md)**: 掌握匿名方法行约束、开放 row 与 `method row requirement`。

### 3. 类型代数与层级
- **[交集与并集](./intersection-union.md)**: 理解类型之间的逻辑组合（与/或关系）。
- **[型变与子类型](./polarity-type.md)**: 深入代数子类型理论，掌握极性（Polarity）、协变与逆变。

### 4. 元编程与高阶抽象
- **[类型函数](./type-function.md)**: 学习如何在编译时通过函数变换类型。
- **[高阶类型 (HKT)](./higher-kinded-types.md)**: 探索对类型构造器的抽象（如 Functor 和 Monad）。
- **[类型级编程](./type-level.md)**: 利用类型系统进行编译时计算与静态证明。

### 5. 尖端特性
- **[依赖类型](./dependent-types.md)**: 模糊值与类型的界限，实现极其严苛的静态约束。
- **[线性类型](./linear-types.md)**: 通过所有权与使用限制，实现精准的资源管理。
- **[效应类型](./effect-type.md)**: 静态追踪副作用，实现纯函数与代数效应的完美结合。

## 核心设计哲学

Valkyrie 的类型系统基于以下原则：

1. **分层优于混层**: `row`、具名 `trait`、`class`、`unite`、`effect` 各自承担不同语义角色。
2. **组合优于误用继承**: 通过 ADT、row 与 trait 组合能力，而不是把所有抽象都塞进 class 层级。
3. **显式优于隐式**: 关键的类型转换、协议满足与效应追踪必须清晰可见。
4. **零成本抽象**: 所有的类型层级在单态化后均不产生运行时开销。
5. **代数一致性**: 类型之间的关系（子类型、交并集）遵循严格的数学格论。

## 四层心智模型

阅读类型系统文档时，建议始终带着下面这组区分：

- `row`：匿名方法约束，只回答“你会不会这些方法”。
- `trait`：具名协议，只回答“你是否满足这个被命名的协议”。
- `class`：对象类型，只回答“你是不是这个类层级里的成员”。
- `unite`：联合类型，只回答“这个值当前属于哪一个分支”。
- `effect`：handler 与能力边界，只回答“你在什么 effect 环境里调用它”。
