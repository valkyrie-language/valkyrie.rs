# 指针与引用 (Pointers & References)

Valkyrie 是一门内存安全的语言，但在底层开发、高性能计算以及外部函数接口 (FFI) 场景下，直接操作内存地址是必不可少的。Valkyrie 提供了一套统一的符号系统来处理原始指针与安全引用。

为了匹配 Valkyrie 的符号美学，我们采用了“圆形”代表安全，“菱形”代表底层的视觉系统：
- **圆形 (Circle)**：安全引用 (Safe References)，受编译器自动生命周期管理。
- **菱形 (Diamond)**：原始指针 (Raw Pointers)，代表底层、危险的操作。

## 引用与修饰符 (References & Modifiers)

Valkyrie 引入了修饰符机制来简化常见的引用操作，使得代码在保持安全性的同时更加简洁。

### 参数修饰符

Valkyrie 的函数参数默认采用**只读引用 (ref)** 传递，这极大地减少了语法噪音：

| 语法 | 说明 |
| :--- | :--- |
| `data: [f64]` | **默认行为**。以只读引用方式传递。 |
| `mut data: [f64]` | 以可变引用方式传递。 |
| `own data: [f64]` | 转移所有权 (Move)。 |

### 类型符号 (Type Symbols)

虽然参数默认是引用，但在定义复杂数据结构（如结构体字段、嵌套容器）时，依然需要符号来明确指出权限：

- **圆形 (Circle)**：显式引用类型，如 `mut T`（可变字段）。
- **菱形 (Diamond)**：原始指针类型，如 `◇T`, `◆T`。

## 解引用操作 (Dereferencing)

Valkyrie 进一步简化了引用的使用，根据类型自动处理解引用：

### 自动解引用 (Automatic Dereferencing)

对于 `class` 类型（引用类型），访问字段或进行赋值时会**自动解引用**，无需显式符号：

```valkyrie
class Variable { value: f64 }

micro update(mut var: Variable) {
    var = Variable(10.0)  # 自动解引用并更新指向的对象
}
```

### 手缀解引用 (Suffix Dereference)

对于 `structure` (值类型) 或原始指针，当需要显式强调解引用行为时，使用对称的后缀符号：

| 权限 | 符号 | 示例 |
| :--- | :--- | :--- |
| **只读** | `.◇` | `ptr.◇` (只读读取) |
| **可变** | `.◆` | `ptr.◆ = val` (可变写入) |

> **注意**：解引用原始指针必须在 `unsafe` 块中进行。

## Null 与 Option 的深度对比

在 Valkyrie 中，虽然 `Option` 和 `null` 都用于处理“缺失”的情况，但它们在语义深度和物理表现上有着本质的区别。

### 1. 物理本质 vs. 逻辑容器
- **`null` (空指针)**：是一个**物理概念**。它代表内存中的 `0` 地址，专门用于原始指针 (`◇T`, `◆T`)。在 `unsafe` 环境下，`null` 是一个合法的（虽然危险的）指针值。
- **`Option⟨T⟩` (可选值)**：是一个**逻辑概念**。它是一个代数数据类型 (ADT)，用于在安全代码中包裹任何类型 `T`。

### 2. 嵌套性：扁平 vs. 层次
这是两者最显著的区别：

- **`Option` 是可嵌套的**：
  `Option⟨Option⟨i32⟩⟩` 在逻辑上具有清晰的层次。
  - `Some(Some(1))`：内部有值。
  - `Some(None)`：外部容器存在，但内部值缺失。
  - `None`：最外层容器就不存在。
  
- **`null` 是不可嵌套的**：
  原始指针的空值是**扁平**的。由于 `null` 仅仅是一个物理地址 `0`，你无法区分“指向 null 的指针”和“空指针本身”。
  - 对于 `◇(◇T)`（指向指针的指针），如果最内层的地址是 `0`，它就是一个 `null`。
  - 不存在类似 `Some(null)` 这种能保留层次结构的物理表示。

### 3. 内存布局优化 (Niche Optimization)
为了兼顾性能与安全，Valkyrie 编译器会对 `Option` 进行物理层面的“空位优化”：
- 对于安全引用 `ref T` 或 `class` 对象，`Option⟨ref T⟩` 的物理大小与 `ref T` 完全一致。编译器利用了有效引用永远不会为 `0` 的特性，将 `None` 直接映射为物理上的 `null`。
- 这种优化使得你在享受 `Option` 的逻辑严密性（可嵌套、类型安全）的同时，获得了与 C 语言原始指针一致的物理效率。

## 迁移指南 (Migration Guide)

### 语法对照表

| 特性 | C 语言 | Rust 语言 | Valkyrie (修饰符/符号) |
| :--- | :--- | :--- | :--- |
| **只读引用** | `const T&` | `&T` | `ref T` |
| **可变引用** | `T&` | `&mut T` | `mut T` |
| **只读指针** | `const T*` | `*const T` | `◇T` |
| **可变指针** | `T*` | `*mut T` | `◆T` |
| **解引用** | `*ptr` | `*ptr` | `ptr.◇` / `ptr.◆` |

### 核心差异

1. **后缀链式调用**：`ptr.◇.field` 比 `(*ptr).field` 更加符合现代链式编程习惯。
2. **权限对称性**：当你看到 `p.◆ = val` 时，你不仅知道是在解引用，还知道这是一个具有“实心/可变”权限的操作。
3. **视觉安全区**：
   - 修饰符 `mut` / `ref`：表示安全，编译器会检查借用规则。
   - 菱形 `◇` / `◆`：表示危险，必须出现在 `unsafe` 环境中。

### 可选的可变指针

```valkyrie
# 表示一个可能不存在的、但如果存在则可修改的内存块
let buffer: ◆u8? = None
```
