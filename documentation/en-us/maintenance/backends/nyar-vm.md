# Nyar VM Maintenance Guide

Nyar VM is Valkyrie's primary runtime and optimization backend, supporting both AOT (Ahead-of-Time) compilation and JIT (Just-in-Time) execution modes.

## Core Compilation Pipeline

After the Valkyrie compiler completes frontend semantic analysis, control is handed to Nyar VM for optimization and generation:

`Source -> AST -> HIR -> UIR (Chomsky) -> Optimized UIR -> Backend Emission`

### 1. HIR -> UIR (Lowering)
- **Responsibility**: Maps language-specific high-level semantics (such as Valkyrie's Effect System, Pattern Matching) to Chomsky's universal intermediate representation (UIR/IKun).
- **Implementation**: `lower_root_to_uir` in [valkyrie-compiler](file:///e:/жЩЃйБНдЉШеМЦ/valkyrie.rs/projects/valkyrie-compiler/src/pipeline/mod.rs).

### 2. UIR Optimization (Chomsky)
- **Responsibility**: Core optimization stage.
- **Technology**: Uses E-Graph-based equality saturation technology.
- **Implementation**: `UniversalOptimizer` in [ProjectChomsky](file:///e:/жЩЃйБНдЉШеМЦ/ProjectChomsky).
- **Advantages**: 
    - Whether AOT or JIT, shares the same optimization logic.
    - Extremely powerful global optimization capabilities (inlining, dead code elimination, constant propagation, etc.).

### 3. AOT Mode
- **Responsibility**: Statically generates executable binary files or WASM modules.
- **Process**: Uses `NyarAot` to drive the optimization process and generates target files through `Gaia` assembler.
- **Implementation**: [nyar-aot](file:///e:/жЩЃйБНдЉШеМЦ/nyar-vm/projects/nyar-aot/src/lib.rs).

### 4. JIT Mode
- **Responsibility**: Generates and executes machine code at runtime based on hotspot code.
- **Process**: Uses `gaia-jit` to dynamically emit instructions to memory and execute them.
- **Implementation**: [nyar-jit](file:///e:/жЩЃйБНдЉШеМЦ/nyar-vm/projects/nyar-jit/src/lib.rs).

## Integration with ProjectChomsky

Nyar VM integrates deeply with Chomsky through the `IKun` interface. Intents generated by the compiler are fed into Chomsky's E-Graph, and after rule rewriting reaches equality saturation, the tree structure with minimum cost is extracted for backend generation.

## Maintenance Recommendations

- **Optimization Rules**: If you need to add new optimization logic, add rewrite rules in `ProjectChomsky`.
- **Backend Adaptation**: If you need to support new instruction sets or platforms, add new assembler adapters in `project-gaia`.
