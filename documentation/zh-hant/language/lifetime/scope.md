# 作用域與自動生命週期插入 (Scope & Auto-insertion)

Valkyrie 摒棄了晦澀且繁瑣的顯式生命週期標註，轉而透過**詞法作用域 (Lexical Scope)**、**參數控制符 (Parameter Qualifiers)** 以及深度的**靜態控制流分析 (Static CFG Analysis)** 來實作 AIFD 全生命週期的自動化管理。

## 判定核心：詞法作用域

在 Valkyrie 中，大括號 `{}` 定義了一個嚴密的物理邊界與執行時間線。編譯器以此為基準，精確編排資源的誕生與消亡。

### 1. 分配與初始化 (Allocate & Initiate)
當變數在作用域內被綁定時，編譯器會立即插入 A 和 I 階段的程式碼。
- **值型別 (`structure`)**：通常表現為堆疊指標 (Stack Pointer) 的移動，實作近乎零成本的分配。
- **引用型別 (`class`)**：觸發受管堆積記憶體的分配請求，並執行建構邏輯。

### 2. 終結與釋放 (Finalize & Delocate)
編譯器會在作用域結束處，或所有可能的提前退出路徑（如 `return`、`break`、`throw`）上，自動注入必要的銷毀邏輯。

```valkyrie
{
    let p = Point { x: 1.0, y: 2.0 }
    if some_condition {
        # 編譯器在此處自動插入 p.finalize() 並安排記憶體回收
        return
    }
}
# 作用域正常退出時，編譯器同樣會自動確保銷毀程式碼的執行
```

## 權屬管理：控制符與預設策略

Valkyrie 引入了控制符（`ref`、`mut`、`own`）來精確定義函式對參數的權屬。為了平衡效能與開發體驗，編譯器為不同型別量身定制了直觀的預設策略。

### 1. 顯式控制符語義

- **引用模式 (`ref` / `mut`)**：代表**借用 (Borrowing)**。不發生所有權轉移。原始作用域在函式呼叫結束後，依然保留對該物件的銷毀責任。
- **所有權模式 (`own`)**：代表**接管 (Taking Ownership)**。發生**移動 (Move)**。物件的生命週期終結責任從呼叫方徹底轉移到被呼叫方。

### 2. 隱式權屬策略 (Implicit Policy)

| 型別分類 | 預設控制符 | 核心設計初衷 |
| :--- | :--- | :--- |
| **Structure** (值型別) | **`own`** | 確保值語義的獨立性，徹底杜絕隱蔽的跨作用域副作用。 |
| **Class** (引用型別) | **`ref`** | 提供透明且自然的共享體驗，符合受管物件的編程直覺。 |

```valkyrie
micro process(d: Data)   # 隱式等價於 (own d: Data)
micro display(p: Person) # 隱式等價於 (ref p: Person)
```

### 3. 設計深度：根除「隱式共享可變性」陷阱

Valkyrie 的預設策略旨在從根本上消除 Python、JavaScript 等動態語言中常見的權屬模糊隱患。

#### 場景 A：共享可變狀態導致的副作用
在傳統受管語言中，簡單的賦值往往會引入危險的隱式共享：
```python
# Python 範例：隱式共享導致資料被意外修改
a = [1, 2]
b = a
a.append(3) 
print(b) # 輸出 [1, 2, 3] —— b 的內容在開發者未察覺的情況下改變了
```

**Valkyrie 的防禦機制**：
1. **針對 Class (引用型別)**：預設採用 **`ref` (唯讀引用)**，嚴格遵循 **讀寫互斥原則**。
    ```valkyrie
    let mut a = [1, 2]
    let b = a        # b 自動獲得 ref [] 權限
    a.push(3)        # 編譯錯誤！無法修改 'a'，因為 'b' 正在以唯讀模式借用它
    ```
2. **針對 Structure (值型別)**：預設採用 **`own` (所有權移動)**。
    ```valkyrie
    let a = Point { x: 1, y: 2 }
    let b = a        # a 的所有權已完整移動至 b，a 在當前作用域失效
    print(a.x)       # 編譯錯誤！'a' 已被標記為失效，無法再次訪問
    ```

#### 場景 B：可變預設參數的狀態殘留
另一個經典的陷阱是函式預設參數在多次呼叫間的狀態持久化：
```python
# Python 範例：預設參數變成了全域狀態
def add_to(item, list=[]):
    list.append(item)
    return list

print(add_to(1)) # [1]
print(add_to(2)) # [1, 2] —— 預設列表竟然在呼叫間被共享了！
```

**Valkyrie 的防禦機制**：堅持 **預設不可變性** 與 **顯式權限授權**。
1. **參數預設唯讀**：無論是 `class` 還是 `structure`，作為參數傳遞時預設均為唯讀。
    ```valkyrie
    micro add_to(item: i32, list: [i32] = []) {
        list.push(item) # 編譯錯誤！'list' 預設僅具唯讀權限
    }
    ```
2. **`mut` 嚴禁攜帶預設值**：若函式確需修改傳入狀態，必須顯式宣告 `mut`，且編譯器強制禁止為此類參數設置預設值。
    ```valkyrie
    # 推薦寫法：呼叫者能一眼識別潛在的副作用
    micro update(mut data: Data) { ... }

    # 編譯錯誤！帶有 'mut' 標識的參數嚴禁擁有預設值，必須由呼叫方顯式傳入
    micro wrong(mut list: [i32] = []) { ... }
    ```

**設計哲學總結**：可變狀態的「所有者」必須始終清晰。Valkyrie 強制要求：**凡涉及狀態修改的操作，必須由呼叫方透過顯式權限進行授權，確保整個權屬鏈條的透明、可追溯與完整。**

---

## 核心決策指南

- **僅需唯讀訪問？** 請使用預設的 `ref`（引用型別）或顯式 `ref`（值型別）。
- **需要原地修改？** 請顯式宣告 `mut` 控制符。
- **需要接管物件或移入容器？** 對於值型別請使用預設行為，對於引用型別請使用 `own`。
- **快速記憶**：值型別 (`structure`) 預設移動，引用型別 (`class`) 預設共享。

## 深度靜態分析：超越詞法邊界

當執行流跨越簡單的詞法邊界時，編譯器會透過高階靜態分析確保生命週期管理的嚴密性。

### 1. 控制流感知 (CFG) 與線性型別追蹤 (Linear Types)

Valkyrie 編譯器透過構建**控制流圖 (Control Flow Graph)**，確保每個擁有所有權的物件在任何可能的執行路徑上都必須被「消費」且僅能被消費一次。

#### 確定性回滾 (Deterministic Unwinding)
在涉及 `?` 錯誤傳播、提前 `return` 或 `throw` 的複雜邏輯中，編譯器會實時計算當前執行點的「存活變數集合 (Live Set)」，並嚴格按照與初始化相反的順序自動插入銷毀程式碼。

```valkyrie
micro process_file() -> Result {
    let f1 = File::open("a.txt")? # 若失敗，無事發生
    let f2 = File::open("b.txt")? # 若失敗，自動插入 f1.finalize()
    let res = compute(f1, f2)?    # 若失敗，自動插入 f2.finalize(), f1.finalize()
    Success
}
```

#### 分支匯合與權屬補償機制
在 `if` 或 `match` 分支中，若不同路徑導致的所有權狀態不一致，編譯器會在匯合點之前進行自動「補償」以對齊狀態：

```valkyrie
let x = Data::new()
if condition {
    consume(x) # 路徑 A: x 的所有權被移動至 consume 函式
} else {
    # 路徑 B: x 仍然存活
    # 編譯器在此處自動插入 x.finalize()
    # 確保匯合點之後，x 在所有路徑上均已失效
}
# 匯合點：x 狀態達成一致（均為失效），後續嚴禁訪問
```

### 2. 逃逸分析 (Escape Analysis)

逃逸分析不僅決定了銷毀的位置，還直接指導了後端的物理分配策略。

#### 逃逸路徑判定
若物件具備以下特徵之一，則標記為「逃逸 (Escaped)」，其生命週期將超越當前作用域：
1. **持久化儲存**：被存入全域變數、單例或長生命週期的容器中。
2. **非同步捕獲**：被 `spawn` 或 `async` 閉包捕獲，進入並行的後台執行流。
3. **權屬返回**：作為函式返回值被 `own` 傳回給呼叫方。
4. **型別擦除**：被轉換為 `any` 動態型別或透過 FFI 介面傳遞給外部語言。

#### 物理優化回饋
- **棧提升 (Stack Promotion)**：若一個 `class` 物件被判定為未逃逸，編譯器會將其從堆積分配提升為堆疊分配。這不僅規避了 GC 壓力，還透過 CPU 快取局部性帶來了巨大的效能提升。
- **標量替換 (SROA)**：若 `structure` 未逃逸且僅局部訪問，編譯器會將其拆解為獨立的標量變數並映射到暫存器，完全消除記憶體訪問。

### 3. 非局部跳轉：Effect 系統與作用域管理

Valkyrie 的代數效應 (Algebraic Effects) 支援靈活的執行流掛起與恢復，這對生命週期管理提出了極高要求。

#### 掛起 (Suspension)
當程式碼執行 `perform` 操作時，當前的執行流和變數環境會被掛起。此時，作用域內的變數**不會觸發** `finalize`，因為它們可能在未來被 Handler 恢復執行。

#### 截斷清理 (Truncation Cleanup)
如果 Handler 決定終止當前的執行流（不再 `resume`），編譯器將識別所有相關的掛起作用域，並按照堆疊回溯順序強制執行所有存活變數的清理邏輯，確保資源不泄露。

#### 作用域快照
對於需要多次恢復執行的效應（如非確定性計算），編譯器會透明地為作用域狀態生成快照，並在每次 `resume` 時恢復 AIFD 模型的中間狀態。
