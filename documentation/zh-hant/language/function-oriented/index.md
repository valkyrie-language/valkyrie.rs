# 函數式程式設計 (Functional Programming)

Valkyrie 是一門深受函數式程式設計思想影響的語言，提供了豐富的函數式特性，使程式碼更加簡潔、可組合且易於測試。

## 核心概念

### 一等公民函數

在 Valkyrie 中，函數是一等公民。這意味著函數可以：
- 賦值給變數
- 作為參數傳遞給其他函數
- 作為其他函數的返回值

### 不變性

Valkyrie 鼓勵使用不可變資料。雖然支援可變性（通過 `mut` 關鍵字），但預設情況下變數是不可變的，這有助於減少副作用並提高並發安全性。

### 純函數

雖然 Valkyrie 允許副作用，但推薦編寫純函數。純函數的輸出僅取決於輸入，且不產生可觀測的副作用。

## 主要特性

### [匿名函數與閉包](./anonymous-functions.md)

匿名函數是沒有名稱的函數，而閉包是可以捕獲其定義環境變數的匿名函數。Valkyrie 提供了簡潔的閉包語法和自動參數推斷。

### [模式匹配](./pattern-match.md)

強大的模式匹配系統允許你根據資料的結構進行分支處理，支援解構元組、陣列、物件以及自定義聯合型別。

### 高階函數

高階函數是接受函數作為參數或返回函數的函數。常見的內建高階函數包括 `map`, `filter`, `fold` 等。

```valkyrie
let numbers = [1, 2, 3, 4, 5]

# 使用高階函數進行組合
let result = numbers
    .filter { $x % 2 == 0 }
    .map { $x * $x }
    .fold(0) { $acc + $item }
```

### 函數組合

通過將簡單函數組合成複雜函數，可以構建高度模組化的系統。

## 優勢

- **簡潔性**: 減少模板程式碼，使邏輯更清晰。
- **可組合性**: 通過組合小型、單一職責的函數來構建複雜功能。
- **可測試性**: 純函數易於進行單元測試，因為它們不依賴外部狀態。
- **安全性**: 減少狀態管理帶來的複雜性和潛在錯誤。
